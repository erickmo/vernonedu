## Quick Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/businesses` | List businesses |
| POST | `/businesses` | Create business |
| GET | `/businesses/{id}` | Get business |
| PUT | `/businesses/{id}` | Update business |
| DELETE | `/businesses/{id}` | Delete business |
| GET | `/businesses/{id}/worksheets` | List worksheets |
| POST | `/businesses/{id}/worksheets` | Create worksheet |
| GET | `/businesses/{id}/worksheets/{ws_id}` | Get worksheet |
| POST | `/businesses/{id}/worksheets/{ws_id}/items` | Add canvas item |
| PUT | `/businesses/{id}/worksheets/{ws_id}/items/{item_id}` | Update item |
| DELETE | `/businesses/{id}/worksheets/{ws_id}/items/{item_id}` | Delete item |
| PATCH | `/businesses/{id}/worksheets/{ws_id}/items` | Bulk update items |
| POST | `/businesses/{id}/worksheets/{ws_id}/submit` | Submit worksheet |

## Authentication

All endpoints require Bearer token.

```
Authorization: Bearer {access_token}
```

**Obtain Token:**
```bash
POST /auth/login
Content-Type: application/json

{
  "email": "student@vernonedu.local",
  "password": "password123"
}
```

## Endpoints

### Business Operations

#### List Businesses
```
GET /businesses?page=1&limit=20&status=draft
Authorization: Bearer {token}
```

**Response (200):**
```json
{
  "data": [ /* businesses */ ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 45,
    "total_pages": 3
  }
}
```

#### Create Business
```
POST /businesses
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "My Startup",
  "description": "Description here"
}
```

**Response (201):**
```json
{
  "data": {
    "id": "bus_123456",
    "name": "My Startup",
    "status": "draft",
    "owner_id": "user_123",
    "created_at": "2026-03-17T14:20:30Z"
  }
}
```

#### Get Business
```
GET /businesses/{business_id}
Authorization: Bearer {token}
```

#### Update Business
```
PUT /businesses/{business_id}
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "Updated Name",
  "description": "Updated Description"
}
```

#### Delete Business
```
DELETE /businesses/{business_id}
Authorization: Bearer {token}
```

## Error Handling

### Error Response
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Validation failed",
    "details": [
      {
        "field": "name",
        "message": "Name is required"
      }
    ],
    "request_id": "req_abc123",
    "timestamp": "2026-03-17T14:20:30Z"
  }
}
```

### Status Codes
| Code | Meaning |
|------|---------|
| 200 | OK |
| 201 | Created |
| 204 | No Content |
| 400 | Bad Request |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not Found |
| 409 | Conflict |
| 422 | Validation Error |
| 429 | Rate Limit |
| 500 | Server Error |

### Error Codes
- `VALIDATION_ERROR` — Validation failed
- `UNAUTHORIZED` — Missing/invalid token
- `FORBIDDEN` — No permission
- `NOT_FOUND` — Resource not found
- `DUPLICATE_ENTRY` — Already exists
- `RATE_LIMIT_EXCEEDED` — Too many requests
- `INTERNAL_ERROR` — Server error

## Examples

### Create Business & Worksheet

```bash
# 1. Create business
curl -X POST https://api.vernonedu.local/v1/businesses \
  -H "Authorization: Bearer token" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Startup","description":"Description"}'

# Response: { "data": { "id": "bus_123456" } }

# 2. Create worksheet
curl -X POST https://api.vernonedu.local/v1/businesses/bus_123456/worksheets \
  -H "Authorization: Bearer token" \
  -H "Content-Type: application/json" \
  -d '{"type":"business-model-canvas","title":"BMC"}'

# Response: { "data": { "id": "ws_abc123" } }

# 3. Add item
curl -X POST https://api.vernonedu.local/v1/businesses/bus_123456/worksheets/ws_abc123/items \
  -H "Authorization: Bearer token" \
  -H "Content-Type: application/json" \
  -d '{
    "section_id":"customer-segments",
    "text":"Startup founders",
    "note":"Jakarta market"
  }'

# 4. Submit worksheet
curl -X POST https://api.vernonedu.local/v1/businesses/bus_123456/worksheets/ws_abc123/submit \
  -H "Authorization: Bearer token" \
  -H "Content-Type: application/json" \
  -d '{"notes":"Ready for review"}'
```

## Validation Rules

**Business:**
- name: Required, 1-100 chars
- description: Optional, max 500 chars

**Worksheet:**
- type: Required, valid type
- title: Required, 1-200 chars

**CanvasItem:**
- text: Required, 1-500 chars
- note: Optional, max 1000 chars
- section_id: Must exist in worksheet
